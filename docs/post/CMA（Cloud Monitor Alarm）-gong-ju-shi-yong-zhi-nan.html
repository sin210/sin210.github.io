📘 CMA（Cloud Monitor Alarm）工具使用指南

基于华为云 Cloud Monitor Alarm（CMA）平台，面向SRE、运维、开发人员的云监控告警系统实操手册

一、什么是CMA？

✅ 定义

CMA（Cloud Monitor Alarm） 是华为云提供的 云监控告警服务，用于实时采集、分析云资源（如ECS、RDS、ELB、VPC等）的性能指标（Metrics），并在指标异常时自动触发告警。

📌 核心能力：
🟢 实时监控
🟢 智能告警
🟢 多通道通知
🟢 告警闭环管理

二、CMA核心功能模块

模块 功能说明
🔹 监控指标采集（Metric Collection） 自动采集云资源的CPU使用率、内存使用率、磁盘IO、网络带宽、数据库连接数等关键指标。
🔹 告警规则配置（Alarm Rule Configuration） 用户可自定义告警阈值、持续时间、触发条件等，实现精准告警。
🔹 告警触发与通知（Alert Trigger & Notification） 当指标超过阈值并持续满足条件时，自动触发告警，并通过多种方式通知责任人。
🔹 告警状态管理（Alert Status Management） 提供“未确认”、“已确认”、“已恢复”、“已关闭”等状态，支持告警生命周期管理。
🔹 告警历史与报表（Alert History & Reports） 可查询历史告警记录，生成告警频率、平均响应时间等运维报表。

三、CMA典型使用场景

场景 使用方式 价值体现
✅ 服务器性能异常告警 配置“CPU使用率 > 90% 持续5分钟”告警规则 防止服务卡顿、宕机
✅ 数据库连接数告警 配置“MySQL连接数 > 200 持续3分钟”告警 避免数据库连接池耗尽
✅ 磁盘空间不足告警 配置“磁盘使用率 > 90%”告警 防止因磁盘满导致服务崩溃
✅ 网络带宽突增告警 配置“出方向带宽 > 90% 峰值持续2分钟”告警 识别DDoS攻击或流量异常
✅ 服务可用性告警（HTTP/HTTPS） 配置“HTTP 5xx错误率 > 1% 持续5分钟”告警 快速发现服务故障

四、CMA操作流程（以华为云CMA为例）

🔧 1. 登录CMA平台

  • 访问地址：https://monitor.cloud.huawei.com
  • 使用企业账号登录（需具备 CMA_Super_reader 权限）

🔍 2. 查看告警列表

  • 进入「告警管理」页面
  • 可按以下条件筛选：
    • 告警状态（未确认 / 已确认 / 已恢复 / 已关闭）
    • 告警级别（P1~P4）
    • 资源类型（ECS、RDS、ELB等）
    • 时间范围

⚙️ 3. 配置告警规则

步骤 1：选择监控对象

  • 点击「创建告警规则」
  • 选择资源类型（如：ECS实例）
  • 选择具体资源（如：ecs-prod-01）

步骤 2：选择监控指标

  • 从列表中选择指标（如：CPU使用率、内存使用率、磁盘IO）

步骤 3：设置告警条件

参数 说明
阈值类型 数值型（如 > 90）、百分比型（如 > 80%）
持续时间 告警需持续多久才触发（如：5分钟）
触发方式 单次触发 / 持续触发（避免频繁告警）
告警级别 P1（紧急） / P2（高） / P3（中） / P4（低）

步骤 4：配置通知方式

  • 选择通知渠道（如：短信、邮件、企业微信、钉钉、Webhook）
  • 指定通知对象（如：SRE团队、值班人、CI Owner）

步骤 5：保存并启用规则

  • 点击「保存并启用」
  • 告警规则立即生效

📢 4. 处理告警（告警闭环流程）

状态 操作说明
🔴 未确认 值班人员收到通知后，需在10分钟内登录CMA平台确认告警是否真实存在
🟡 已确认 确认为真实告警，需立即启动故障排查流程
🟢 已恢复 问题已解决，指标恢复正常，点击「恢复」按钮
🔴 已关闭 告警已处理完毕，可手动关闭或设置自动关闭（如：72小时未恢复自动关闭）

✅ 重要提醒：
所有告警必须走完“确认 → 排查 → 恢复 → 关闭”闭环流程，避免“告警堆积”和“责任不清”。

五、通知方式与集成

通知方式 说明 适用场景
📱 短信 实时推送，适合P1级告警 紧急故障
📧 邮件 可附带详细告警信息 普通告警
💬 企业微信 支持@人、群发、一键跳转 团队协作
📩 钉钉 支持群机器人、自定义卡片 中大型团队
🔗 Webhook 支持与自研系统对接（如：自研告警中心、工单系统） 自动化运维

✅ 建议：
对P1级告警，建议同时启用 短信 + 企业微信 + Webhook 三重通知，确保第一时间响应。

六、权限管理规范

权限等级 权限说明 适用角色
Super_reader 超级只读权限 —— 可查看所有告警、规则、通知记录，但不能修改或删除 所有SRE、运维、开发人员
Reader 普通只读权限 —— 可查看本项目/本部门下的告警信息 部门内普通成员
Editor 编辑权限 —— 可创建、修改、删除告警规则，配置通知方式 告警规则管理员
Super_admin 超级管理员 —— 拥有所有权限，可管理用户权限、系统配置 CMA平台管理员

✅ 重要提醒：
所有人员在使用CMA前，必须申请并获得 CMA_Super_reader 权限，否则无法正常开展工作。

七、最佳实践建议

实践 说明
✅ 告警规则“少而精” 避免“告警泛滥”，每条规则应聚焦一个关键业务指标
✅ 设置合理的持续时间 避免瞬时抖动导致误告警，建议P1级告警设置为“持续5分钟”
✅ 分级告警管理 根据影响范围和严重程度设置P1~P4级别，匹配不同响应策略
✅ 建立告警自愈机制 对于常见、可预测的告警（如：磁盘满），可通过自动化脚本实现“自愈”
✅ 定期清理过期告警规则 每季度组织一次告警规则稽查，清理不再适用的规则

八、常见问题与解决方案（FAQ）

问题 原因 解决方案
❓ 收到大量重复告警 告警规则持续时间设置过短或未设置
❓ 告警未触发，但指标已超标 告警规则未启用，或通知对象配置错误
❓ 无法查看某条告警的详细信息 该告警所属资源的权限不足
❓ 告警恢复后仍显示“未恢复” 未手动点击“恢复”按钮，或系统未正确识别指标恢复状态

九、附录：常用快捷操作与技巧

快捷操作 说明

Shift + 点击 批量选择多个告警
F5 刷新当前页面
