## 1. 术语定义

| 名称 | 定义 |
|------|------|
| **巡检** | 指对云计算环境（包括：机房物理环境、云平台基础设施、云服务组件、业务应用系统等）进行的定期或不定期检查活动，旨在识别潜在隐患与风险，提出优化建议，提前排除问题，防止服务中断或数据丢失。 |
| **巡检转移** | 指共享中心尚未承接的巡检作业，从原业务团队转移至共享中心进行集中管理与执行的过程。 |
| **巡检更新** | 指已由共享中心承接的巡检项，在内容、标准、规则或处置策略等方面进行修订或优化，包括但不限于异常判断标准变更、风险影响等级调整、修复建议更新等。 |
| **巡检风险** | 指在巡检任务执行过程中，依据既定标准发现的客户现网或内部环境中的潜在隐患或风险点，可能影响系统稳定性、服务可用性或数据安全性。 |

## 2. 角色与职责

| 角色名称 | 职责说明 |
|-----------|------------|
| **业务转移诉求方代表** | <ul><li>发起巡检作业转移或更新申请，按要求完成业务成熟度自评与评估报告编制。</li><li>确认转移范围，结合当前业务运行模式，提出人员人力基线建议。</li><li>与共享中心PM协同对齐业务转移范围及联合运作机制，提供人员支持，包括培训、首次作业实操指导等。</li><li>作为巡检基线Owner，依据版本管理机制定期维护和更新巡检基线，并同步至交付团队与产品部门。</li><li>若巡检风险无法按期解决，作为技术攻关负责人参与风险闭环处理。</li></ul> |
| **共享中心PM** | <ul><li>受理巡检作业转移或更新申请，审阅业务转移方提交的成熟度自评报告，出具初审意见。</li><li>在技术评审通过后，组织完成业务评审，与业务方共同确认转移范围与运作机制。</li><li>在新作业灰度运行期间，组织联合运作与人力切换，逐步实现共享中心独立运作能力。</li><li>在新业务具备稳定独立运作能力后，输出装载总结报告，正式完成业务装载承接。</li></ul> |
| **共享中心TD** | <ul><li>在申请初审通过后，主导新业务的技术风险评估工作，涵盖作业SOP评审、自动化能力成熟度评估、作业风险识别与分级评估等。</li></ul> |
| **巡检执行人** | <ul><li>严格按照评审通过的作业SOP执行巡检任务，准确识别并记录巡检风险，清晰描述风险影响范围与严重程度，提出可操作的修复建议。</li><li>对于无法独立解决的巡检风险，应立即按流程逐级上报，直至问题得到有效响应与处置。</li></ul> |

## 5 巡检作业执行要求

### 5.1 巡检作业执行要求
• **严格遵循标准**  
  禁止遗漏巡检项。若作业环境不满足或存在风险，需及时提出。  
  巡检SOP（标准操作流程）必须经过三方联合评审，新增或刷新SOP需重新评审（能力中心/共享中心各域TD/巡检SE）。

• **明确基线范围**  
  唯一有效的巡检基线为《共享中心飞行员手册》中“5 巡检作业--查询类SOP清单”。SOP仅用于查询，不涉及变更操作。

• **风险通报机制**  
  工具异常导致的误报信息需在保障群组通报进展。  

---

### 5.2 巡检作业执行质量要求
• **0漏检/误检导致现网P4+事件**  
  按时完成巡检，确保风险及时发现并解决。  
  **惩罚机制**：因漏检/误检引发P4+事件者，将追责。

• **时效性要求**  
  巡检及时完成率不低于 **99%**：  
  - 巡检时间窗：提前10分钟（集中保障30分钟），延后20分钟。  
  - 离岗、处理其他事务时需提前完成交接或安排备份人员。

• **结果准确性**  
  巡检反馈结果准确率不低于 **99%**：  
  - 客户信息、时间、指标结果等需严格校对。  
  - 错漏信息需相互提醒并及时撤回，超时后补发修正结果。

   - 所有异常均需登记至巡检风险记录表。

## 7 巡检作业稽查规则和稽查执行标准

### 7.1 稽查规则说明

| 序号 | 稽查项目                     | 稽查方法                                                                 | 判断标准                                                                 | 业务影响                            |
|------|------------------------------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|-------------------------------------|
| 1    | 巡检任务是否按时完成         | - 随机稽查3家客户3天的巡检作业<br>- 已晚检/漏检视为违约                  | - 巡检及时完成率99%<br>- 0漏检导致现网事件                              | 影响及时性与现网安全                |
| 2    | 巡检结果与实际是否相符       | - 核对反馈时间、群组、范围及结果                                         | - 禁止谎报问题，影响事故处理<br>- 推动工具改进                           | 涉及运维红线                        |
| 3    | 是否按SOP完成巡检作业        | - 人工/自动化稽查配置项是否齐全                                          | - 遗漏巡检项或判断不准                                                  | 巡检结果不可靠                      |
| 4    | 巡检异常风险是否正确记录     | - 核查登记表与实际风险项比对                                             | - 风险需完整记录<br>- 多项风险需分条记录<br>- 重复风险无需重复记录         | 降低风险遗漏率，优化工具            |
| 5    | 巡检异常风险闭环是否合理     | - 3天内核查风险更新状态与闭环时效                                         | - **关闭**：风险彻底消除<br>- **挂起**：TAM确认需客户/SRE解决<br>- **风险预警**：未闭环需预警 | 闭环时效性、人力优化                |

&gt; **注**：稽查争议处理流程：  
&gt; 1. 争议问题上升巡检SE处理；  
&gt; 2. 巡检SE无法达成一致，上报华为主管决策。
