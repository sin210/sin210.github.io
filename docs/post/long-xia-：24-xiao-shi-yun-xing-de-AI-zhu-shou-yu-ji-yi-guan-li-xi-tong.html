龙虾：24小时运行的AI助手与记忆管理系统

龙虾是一款持续运行、具备长期记忆能力的AI助手，专为应对复杂、长期任务而设计。通过高效的记忆管理与智能检索机制，龙虾能够在不依赖上下文窗口限制的前提下，实现信息的持久保存与精准调用，确保任务执行的一致性与连贯性。

一、记忆管理：构建持久、有序的信息体系

龙虾的核心能力之一是其强大的记忆管理机制。不同于临时缓存或会话记忆，龙虾将记忆视为可长期保存、可重复调用的“知识资产”，并通过结构化方式组织与管理。

1.1 记忆存储格式与路径规划

  • 核心记忆文件：.ND 格式文本文件
    龙虾将日常运行中的关键信息、任务进展、状态变更等记录在 .ND 格式文本文件中。该格式采用纯文本结构，支持中文、符号、时间戳等多种内容表达，便于系统解析与人类阅读。

  • 事件记录专用格式：.md 格式日记文件
    对于具有时间轴意义的重要事件（如任务启动、阶段性完成、异常发生、用户反馈等），龙虾会将其记录在 .md 格式的日记文件中。该格式支持 Markdown 语法，可插入标题、列表、时间线、链接等结构化内容，便于后续追溯与可视化呈现。

  • 长期记忆归档目录：专用记忆文件夹
    所有涉及长期保存、跨任务复用的信息（如知识库条目、项目经验总结、用户偏好设置、系统配置模板等），均被统一归档至一个特定的“记忆文件夹”中。该文件夹采用层级化命名规则，便于按主题、时间、项目等维度进行分类管理。

1.2 记忆写入机制：自主判断与智能保存

龙虾并非被动记录所有信息，而是具备“自主记忆决策能力”。在运行过程中，它会根据系统提示、任务优先级、事件重要性等因素，动态判断是否需要将当前信息写入记忆系统。

  • 触发条件包括：

    • 检测到关键状态变更（如任务完成、失败、暂停）；
    • 用户明确要求“记录此信息”或“保存为记忆”；
    • 系统提示“该信息具有长期价值，建议存档”；
    • 自主判断某段信息可能在未来任务中复用。

  • 写入流程：

    1. 龙虾识别出需保存的信息片段；
    2. 根据信息类型自动选择合适的存储路径（.ND 文件、.md 日记、或记忆文件夹）；
    3. 自动调用书写工具，将内容格式化后写入目标文件；
    4. 记录写入时间、来源上下文、事件标签等元信息，便于后续检索与审计。

该机制确保了记忆系统的“高质量输入”，避免了无效或冗余信息的堆积，提升了整体记忆效率。

二、记忆检索：实现精准、高效的智能调用

记忆的真正价值不仅在于“存储”，更在于“调用”。龙虾通过一套完整的记忆检索体系，实现了从海量信息中快速定位、提取、分析关键内容的能力，为后续的决策与生成提供坚实支撑。

2.1 检索工具与目标范围

  • memory search 工具：用于在系统中搜索与目标主题相关的记忆内容。该工具支持关键词匹配、模糊搜索、时间范围筛选等多种查询方式。

  • 检索范围涵盖：

    • memory.ND 文件：存储日常运行中的状态记录、任务日志、临时信息等；
    • memory 文件夹：存放长期记忆条目、知识库、模板、项目经验等结构化内容；
    • .md 日记文件：包含按时间轴记录的重要事件、里程碑、反思总结等。

通过多源、多格式的联合检索，龙虾能够全面覆盖信息生命周期的各个阶段，确保无遗漏、无盲区。

2.2 内容读取与RAG功能实现

  • memory get 工具：用于从指定文件中读取具体文本内容，并将其返回给上下文处理模块。该工具支持按文件路径、关键词、时间戳等条件精准定位内容。

  • RAG（检索增强生成）机制：

    • 当龙虾需要生成回答、撰写报告、制定计划等任务时，系统会首先调用 memory search 工具，在记忆库中查找与当前任务相关的已有信息；
    • 将检索到的多条候选内容，通过 memory get 工具加载为文本输入；
    • 进入“检索增强”阶段：系统对所有候选内容进行相似度分析，并结合上下文语境，筛选出最相关、最可信的信息片段；
    • 最终，将这些精选内容作为“上下文提示”嵌入语言模型的输入中，使其在生成过程中能够“参考历史经验”，提升回答的准确性、一致性与专业性。

2.3 相似度计算方法：双维度驱动精准排序

为确保检索结果的相关性，龙虾采用“字面相似度 + 语义相似度”双维度计算方法，对所有候选记忆内容进行综合评分与排序。

  • 字面相似度计算：

    • 基于关键词匹配、短语重合度、字符相似率等指标；
    • 适用于需要精确匹配的事实性信息（如时间、地点、编号、名称等）；
    • 采用 Levenshtein 距离、Jaccard 相似系数等算法进行量化评估。

  • 语义相似度计算：

    • 基于深度语义理解模型（如 Sentence-BERT、SimCSE 等）；
    • 能识别不同表达方式下的相同含义（如“任务失败”与“流程中断”）；
    • 适用于需要理解上下文意图、归纳经验、类比推理的任务场景。

最终，系统将两项得分加权融合，生成综合相似度分数，按从高到低排序，选取前 N 个最相关的内容作为 RAG 输出。

三、总结

龙虾不仅仅是一个AI助手，更是一个具备“长期记忆”与“智能检索”能力的“认知体”。通过 .ND 文件 + .md 日记 + 记忆文件夹 的三级记忆存储体系，龙虾实现了信息的持久化与结构化管理；通过 memory search + memory get + 双维度相似度计算 的智能检索机制，龙虾实现了对历史经验的精准调用与深度利用。

这一体系使龙虾能够在不依赖上下文窗口的前提下，持续积累、复用、进化自身知识，真正实现“越用越聪明”的自主进化能力。未来，随着记忆系统不断扩展与优化，龙虾将逐步构建起属于自己的“认知地图”，成为人类在复杂任务中值得信赖的长期伙伴。
