📘 CMC（Cloud Monitor Center）工具使用指南

基于华为云 Cloud Monitor Center（CMC）平台，面向SRE、运维、开发人员的云监控中心实操手册

一、什么是CMC？

✅ 定义

CMC（Cloud Monitor Center，云监控中心） 是华为云提供的 统一监控可视化平台，用于集中展示、分析、管理全栈云资源（ECS、RDS、ELB、VPC、Kubernetes、微服务等）的运行状态与性能指标。

📌 核心定位：
🟢 一站式监控视图
🟢 全链路性能追踪
🟢 多维度指标分析
🟢 与CMA告警系统无缝联动

二、CMC核心功能模块

模块 功能说明
🔹 全局监控仪表盘（Dashboard） 提供组织/项目/环境级别的统一监控视图，支持自定义可视化组件（图表、指标卡、拓扑图等）。
🔹 资源监控（Resource Monitoring） 实时查看单个或多个云资源（如ECS、RDS）的CPU、内存、磁盘、网络等关键指标。
🔹 链路追踪与调用分析（Tracing & Call Analysis） 支持微服务架构下的请求链路追踪，可视化展示请求在各服务间的流转路径、耗时分布、错误率等。
🔹 日志与指标联动分析（Log-Metric Correlation） 支持将监控指标与关联日志（如应用日志、访问日志）进行双向联动，快速定位故障根因。
🔹 告警联动与事件中心（Alert & Event Integration） 与CMA告警系统深度集成，支持在CMC中直接查看告警事件、处理告警、查看影响范围。
🔹 自定义监控模板与共享（Custom Template & Sharing） 支持创建、保存、共享监控模板（如“Web服务监控模板”、“数据库性能监控模板”），提升团队协作效率。

三、CMC典型使用场景

场景 使用方式 价值体现
✅ 全栈资源状态一目了然 在仪表盘中查看组织内所有ECS、RDS、ELB等资源的运行状态与关键指标 提升全局掌控力
✅ 微服务调用链路故障定位 在“链路追踪”模块中查看某次请求的完整调用路径，识别耗时瓶颈或异常服务 缩短MTTR（平均修复时间）
✅ 应用性能瓶颈分析（APM） 查看某服务的请求响应时间、TPS、错误率等指标，结合日志分析性能瓶颈 优化系统架构
✅ 告警事件快速响应 在“事件中心”中查看CMA触发的告警，直接跳转至相关资源监控页，启动排查流程 提高告警处理效率
✅ 新系统上线前性能基线评估 在上线前，通过CMC采集系统在高负载下的关键指标，建立性能基线 预防上线后性能问题

四、CMC核心操作流程（以华为云CMC为例）

🔧 1. 登录CMC平台

  • 访问地址：https://monitor.cloud.huawei.com
  • 使用企业账号登录（需具备 CMC_Super_reader 权限）

📊 2. 进入仪表盘（Dashboard）

  • 首页默认展示“组织级”或“项目级”仪表盘
  • 可通过顶部导航栏切换不同维度的视图：
    • 🌐 全组织视图
    • 📦 项目/环境视图（如：生产/预发/测试）
    • 🖥️ 单资源视图（如：某台ECS服务器）

📈 3. 查看资源监控指标

  • 在仪表盘中点击某个资源（如：ecs-prod-01）
  • 进入资源监控详情页，查看：
    • 实时指标曲线（CPU、内存、磁盘IO、网络带宽）
    • 指标趋势图（按小时、天、周查看）
    • 指标统计信息（最大值、最小值、平均值、当前值）

🔗 4. 使用链路追踪功能（Tracing）

  • 进入「链路追踪」模块
  • 输入请求ID或服务名，查找特定请求
  • 查看请求的完整调用路径，包括：
    • 各服务的调用耗时（毫秒）
    • 调用状态（成功/失败）
    • 错误信息（如：超时、500错误）
  • 点击任一节点，可查看该服务的监控指标与日志

📝 5. 日志与指标联动分析

  • 在资源监控页，点击某条异常指标曲线
  • 选择「关联日志」，系统将自动加载该时间段内的相关日志
  • 通过关键词搜索（如：error、timeout），快速定位问题日志
  • 可将日志片段直接分享给开发人员或SRE团队

🛠️ 6. 查看告警事件与处理

  • 进入「事件中心」模块
  • 查看所有来自CMA系统的告警事件
  • 可按以下条件筛选：
    • 告警级别（P1~P4）
    • 告警类型（CPU过高、磁盘满、网络异常等）
    • 时间范围
  • 点击任一条告警，可：
    • 查看影响范围（哪些资源受到影响）
    • 查看告警详情与触发条件
    • 直接跳转至相关资源监控页
    • 手动确认或恢复告警

五、仪表盘自定义与模板管理

功能 说明
✅ 创建自定义仪表盘 点击「新建仪表盘」，添加图表、指标卡、拓扑图等组件，按需布局
✅ 保存与命名仪表盘 为仪表盘命名（如：“生产环境监控大盘”），并选择所属项目或环境
✅ 共享仪表盘 可将仪表盘分享给指定团队成员或角色（如：SRE团队、开发组）
✅ 使用预置模板 CMC提供多种预置监控模板（如：Web服务、数据库、Kubernetes集群），可一键应用

✅ 建议：
每个项目/环境应建立专属的“监控仪表盘”，并定期优化布局与指标展示。

六、权限管理规范

权限等级 权限说明 适用角色
Super_reader 超级只读权限 —— 可查看所有仪表盘、资源监控、链路追踪、事件中心内容 所有SRE、运维、开发人员
Reader 普通只读权限 —— 可查看本项目/本部门下的监控数据 部门内普通成员
Editor 编辑权限 —— 可创建、修改、删除自定义仪表盘与模板 监控管理员、SRE负责人
Super_admin 超级管理员 —— 拥有所有权限，可管理用户权限、系统配置 CMC平台管理员

✅ 重要提醒：
所有人员在使用CMC前，必须申请并获得 CMC_Super_reader 权限，否则无法正常开展工作。

七、最佳实践建议

实践 说明
✅ 仪表盘“按需设计” 不同团队关注点不同：SRE关注资源状态，开发关注服务性能，领导关注整体健康度
✅ 启用“链路追踪”常态化监控 对关键业务服务开启链路追踪，实时监控请求耗时、错误率，提前发现性能瓶颈
✅ 建立“监控基线”与“异常阈值” 在系统上线前采集正常负载下的指标数据，作为“基线”；设置合理的“异常阈值”以避免误报
✅ 定期清理过期或无用的仪表盘 每季度组织一次监控资产盘点，清理不再使用的仪表盘与模板
✅ 与CI/CD流水线联动 在CI/CD流水线中集成CMC监控，上线后自动采集性能指标，验证上线质量

八、常见问题与解决方案（FAQ）

问题 原因 解决方案
❓ 无法查看某资源的监控数据 该资源未启用监控采集，或权限不足
❓ 链路追踪中找不到某次请求 该请求未开启链路追踪，或追踪ID已过期
❓ 仪表盘显示延迟或空白 网络连接不稳定或浏览器缓存异常
❓ 告警事件未同步到CMC CMA与CMC未正确集成，或事件同步服务异常

九、附录：常用快捷操作与技巧

快捷操作 说明

Shift + 点击 批量选择多个资源或事件
F5 刷新当前页面
