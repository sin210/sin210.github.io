# 📘 CLS（Cloud Log Service）工具使用指南（上篇）

> 基于华为云 Cloud Log Service（CLS）平台，面向SRE、运维工程师、开发人员、安全团队的**云上日志统一管理与智能分析实操手册**

---

## 一、什么是CLS？

### ✅ 定义
Cloud Log Service（简称 CLS）是华为云提供的一站式日志管理平台，旨在帮助用户实现对来自各类系统与应用的日志数据进行集中采集、统一存储、高效查询、智能分析与实时告警。

CLS 支持从云服务器、容器服务、数据库、负载均衡、安全防护设备、前端应用、移动应用等多类来源采集日志，适用于大规模、高并发、多系统的复杂运维环境。

---

## 二、CLS核心功能模块

| 模块 | 功能说明 |
|------|----------|
| 🔹 **日志采集** | 支持通过日志采集器、API 接口、消息队列、对象存储等多种方式，从不同系统中自动获取日志数据。 |
| 🔹 **日志存储与索引** | 所有日志自动按时间分区存储，并建立全文检索与字段索引，确保查询响应时间在毫秒级。 |
| 🔹 **日志查询与分析** | 提供强大的查询能力，支持按时间、日志级别、关键词、字段值等条件进行灵活筛选，并可对日志数据进行聚合、分组、排序等分析操作。 |
| 🔹 **日志告警** | 可基于日志内容设置动态告警规则，当检测到异常日志、错误频发、访问行为异常等情况时，系统将自动触发告警，并通过邮件、企业微信、短信、Webhook 等方式通知相关人员。 |
| 🔹 **可视化报表与看板** | 支持创建自定义看板，集成柱状图、折线图、饼图、热力图等可视化图表，直观展示日志分析结果，便于监控系统状态、分析用户行为、评估安全风险。 |

## 三、CLS典型使用场景

| 场景 | 使用方式 | 价值体现 |
|------|-----------|-----------|
| ✅ **应用系统故障排查** | 将应用日志通过采集器上传至CLS，利用查询功能快速定位错误日志，分析错误原因。 | 显著缩短故障排查时间，提升系统可用性。 |
| ✅ **Web访问行为分析** | 采集网站访问日志，分析访问来源、请求路径、访问频率、响应时间等，识别高频访问与异常行为。 | 帮助优化网站性能，识别潜在的DDoS攻击或爬虫行为。 |
| ✅ **安全事件告警与响应** | 配置告警规则，当检测到数据库删除操作、文件删除、权限变更等高危行为时，立即发送告警通知。 | 实现安全事件的主动发现与快速响应，降低安全风险。 |
| ✅ **用户行为分析与运营决策** | 采集移动App或Web前端的用户行为日志，分析用户登录、点击、浏览、下单、支付等行为路径，生成用户画像与转化漏斗。 | 支持精细化运营，优化产品功能与用户体验。 |
| ✅ **合规审计与等保2.0支持** | 将所有系统操作日志统一采集，生成完整的审计记录，满足等保2.0“安全审计”要求，为合规检查提供依据。 | 降低合规风险，确保系统符合国家信息安全标准。 |

## 四、CLS核心操作流程（以华为云CLS为例）

### 🔧 1. 登录CLS控制台
通过浏览器访问华为云CLS控制台，使用企业账户登录后即可进入主界面。

### 📥 2. 创建日志主题
在控制台中点击“新建主题”，填写主题名称（如：`web-access-logs`）、描述信息，并选择日志来源类型（如：ECS、K8s、API等）。随后根据日志格式配置相应参数，完成创建。

### 🛠️ 3. 配置日志采集方式
- **通过采集器采集**：在服务器或容器中安装CLS采集器，配置需要采集的日志文件路径，启动采集器后，日志将自动上传至CLS。
- **通过API推送日志**：在应用系统中集成CLS提供的API接口，将日志数据以结构化格式推送到指定主题。
- **通过第三方系统接入**：支持通过Kafka、S3、MinIO等中间系统将日志数据接入CLS，适用于复杂架构场景。

> ✅ **建议**：优先使用采集器方式，具备自动发现、断点续传、日志压缩、加密传输等优势，稳定性与安全性更高。

## 五、日志采集与格式规范（关键实践）

### ✅ 1. 统一日志格式
建议所有系统输出日志时采用统一的结构化格式，如JSON格式，以便于统一解析与分析。

### ✅ 2. 必须包含的关键字段
为保证日志的可读性与分析效率，建议日志中包含以下核心字段：

- **时间戳**：记录日志生成的精确时间（建议使用标准时间格式）。
- **日志级别**：标明日志的严重程度，如：调试、信息、警告、错误、致命。
- **服务名称**：标识生成该日志的服务或模块名称。
- **追踪ID**：用于跨服务、跨系统的链路追踪，便于定位问题发生路径。
- **日志内容摘要**：对事件或操作的简要描述，便于快速理解日志含义。

> ✅ **提示**：避免在日志中直接写入敏感信息（如密码、身份证号、银行卡号等），防止信息泄露。

## 六、日志查询与分析基础操作（入门必学）

### ✅ 1. 进行基础查询
在CLS控制台中打开“日志查询”功能，选择目标日志主题后，可直接输入查询条件，系统将自动返回匹配的日志条目。

### ✅ 2. 使用条件过滤
可通过设置多种条件来筛选日志，例如：
- 按日志级别筛选（如：只查看错误日志）
- 按时间范围筛选（如：查看过去一小时的日志）
- 按关键词或字段值筛选（如：查找包含“timeout”或“500”状态码的日志）

### ✅ 3. 提取关键字段进行分析
在查询结果中，可选择特定字段（如IP地址、请求路径、用户ID）进行查看与分析，便于识别访问模式或异常行为。

### ✅ 4. 进行聚合分析
支持对日志数据进行分组统计与聚合分析。例如：
- 统计每个IP地址的访问次数
- 统计不同服务模块的错误日志数量
- 按时间维度分析日志生成趋势

此类分析有助于发现系统瓶颈、识别高风险行为、评估系统健康状况。
