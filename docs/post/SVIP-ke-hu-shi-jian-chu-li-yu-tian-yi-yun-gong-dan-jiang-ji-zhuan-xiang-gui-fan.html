SVIP客户事件处理与天翼云工单降级专项规范

一、SVIP客户定义与事件处理机制

SVIP客户为公司战略级客户，享有直达总裁级别的专属服务特权。为切实提升其服务满意度与业务连续性保障能力，针对SVIP客户提出的各类事件请求，制定以下专项处理规范：

1. 事件提单级别与处理路径

  • 非咨询类请求（如系统故障、资源异常、服务中断等）：统一提报为 P4A+ 事件。
    P4A+事件由能力中心团队直接受理，无需逐级上升，确保资源快速调配与问题闭环。

  • 咨询类请求（如产品使用指导、配置建议、方案咨询等）：统一提报为 P5 事件。
    P5事件由TAM或一线客服团队优先处理，承诺在2小时内完成初步响应与处理；若无法解决，须立即升级至CIM系统，由共享中心或能力中心团队接手处理，杜绝响应延迟。

2. 高优先级响应与应急机制

  • 所有SVIP客户事件，无论提单级别如何，均须优先于普通事件工单处理，确保响应速度与资源投入匹配其战略地位。
  • 若事件存在业务影响，必须统一拉起WarRoom，由二级RL（责任领导）全程参与指挥，协调多团队联动处置，确保问题快速定位与恢复。
  • 事件风险须第一时间上报至域内SL（服务负责人），并同步通知客户及提单人，确保信息透明、响应及时。

3. 关单定级标准

SVIP客户事件的关单定级必须严格依据客户等级执行：

  • 故障类事件：若客户等级满足 V5企业级及以上，则事件定级为 P4A，必须闭环；
    若客户等级低于V5企业级，可降级为 P4 闭环，但必须在事件单中明确说明降级原因及客户等级依据。

  • 所有关单必须在规定时间内完成定级与关闭，并保留完整的处理记录，确保可追溯、可审计。

二、天翼云工单降级专项规范

为规范天翼合营云在现网事件的降级工作，确保客户问题高效处理、服务水平持续稳定，特制定本专项规范。

1. 目的

规范天翼合营云在现网事件的降级流程，确保在不影响业务的前提下，快速恢复服务，保障服务质量和可用性满足SLA要求。同时，强化过程管控，杜绝私自降级行为。

2. 适用范围

适用于天翼云现网事件，包括但不限于大云、荣耀、HCSO等区域的工单事件。

3. 降级稽查点与登记要求

针对天翼云大云工单，降级前必须完成以下操作：

  • 登记要求：所有降级操作前，必须先在指定在线登记表中完成登记。
    登记地址：https://onebox.huawei.com/p/f75036e01e4205e43f4ea1aff6bbf281

  • 审批流程：登记后需等待天翼云侧审批通过，方可执行降级操作。
    若工单降级操作未在登记表中记录，或未获审批通过，将视为“私自降级”行为，按严重违规处理。

  • 信息填写要求：

    • 降级工单可不回填同意人信息，但必须在登记表中有完整记录且审批通过。
    • 特殊情况：HCSO和荣耀局点（贵州、咸阳、华北）在降级时仍需填写同意人信息，要求不变。

4. 各类事件的降级流程

4.1 大云事件单（未闭环或闭环未超一个月）

  • 适用条件：工单未闭环，或已闭环但时间未超过一个月（CIM可修改）。
  • 操作流程：
    1. 在工单描述第一行添加电信侧工单号：【xxxxxxxxxx】。
    2. 登记至指定在线链接：https://onebox.huawei.com/p/c5aa62864d5f731a50654612e3736249。
    3. 等待天翼云侧审批通过。
    4. 每星期五统一提交降级申请至电信侧。

4.2 荣耀团队、HCSO区域事件单

  • 适用条件：工单已生成，需进行降级操作。
  • 操作流程：
    1. 与提单人充分沟通，获得其明确同意。
    2. 在工单中填写以下信息：
      • 【业务影响】：说明是否影响客户业务。
      • 【升降级理由】：详细说明降级原因。
      • 【升降级同意人】：必须填写具体同意人姓名与职务。
    3. 完成填写后方可执行降级操作。

4.3 工单关闭超过一个月（CIM不可更改，需GOC更改）

  • 适用条件：工单已关闭超过一个月，系统已锁定，无法通过CIM修改。
  • 操作流程：
    1. 通过邮件方式提交升降级申请。
    2. 邮件发送前，申请人必须确保工单信息完整，包括：
      • 事件类别
      • 出现原因
      • 解决方案
      • 问题单关联
    3. 邮件发送至指定审批人，由GOC在后台进行CIM等级更改。

    • 若不影响业务：进入升降级通道，按流程执行。

  • 若电信侧对降级申请存在异议，不认同可联系王若维（天翼云接口人）协调处理。

三、执行要求与注意事项

  1. 严禁私自降级：所有降级操作必须经过登记与审批流程，违反者按公司制度严肃处理。
  2. 信息填写必须完整：尤其是“升降级同意人”“业务影响”等关键字段，不得遗漏或虚假填写。
  3. 强化跨团队协同：一线、TAM、共享中心、能力中心、GOC、天翼云侧等各方须通力合作，确保流程顺畅、责任清晰。
  4. 定期复盘与优化：各团队应定期对SVIP客户事件与天翼云工单降级情况进行复盘，识别流程堵点，持续优化服务质量。
