📘 CAC（Cloud Auto Change）工具使用指南

基于华为云 Cloud Auto Change（CAC）平台，面向SRE、DevOps、运维工程师的云上自动化变更管理实操手册

一、什么是CAC？

✅ 定义

CAC（Cloud Auto Change，云自动化变更） 是华为云提供的 智能化、可审计、可回滚的云资源自动化变更平台，用于标准化、安全地执行各类云资源的变更操作（如：扩容、降配、配置更新、镜像升级、灰度发布等）。

📌 核心定位：
🟢 自动化执行 —— 减少人工误操作
🟢 全链路可审计 —— 变更全过程留痕
🟢 审批与熔断机制 —— 保障变更安全
🟢 自动回滚与补偿 —— 降低变更风险

二、CAC核心功能模块

模块 功能说明
🔹 自动化变更流程引擎（Workflow Engine） 支持可视化编排变更流程，包含“预检查 → 执行 → 验证 → 回滚”等标准步骤。
🔹 变更策略配置中心（Policy Center） 支持配置变更策略，如：执行时间窗口（如：00:00–06:00）、最大并发数、是否强制执行等。
🔹 审批与分级授权（Approval & Role-Based Access） 支持多级审批流程（如：开发 → SRE → 运维主管），并基于角色控制操作权限。
🔹 变更执行与日志追踪（Execution & Audit Log） 所有变更操作均自动记录执行日志，包括：操作人、时间、执行结果、耗时、错误信息等。
🔹 变更失败自动回滚（Auto Rollback） 当变更过程中任一环节失败，系统将按照预设策略自动触发回滚流程，恢复至变更前状态。
🔹 变更影响范围分析（Impact Analysis） 在变更前，系统可自动分析该变更对上下游服务、依赖资源、SLA指标的影响，提示潜在风险。
🔹 变更报告与统计分析（Report & Analytics） 提供变更成功率、平均耗时、失败原因分布等统计报表，支持按团队、项目、资源类型维度分析。

三、CAC典型使用场景

场景 使用方式 价值体现
✅ ECS实例扩容/缩容 配置“凌晨00:00~06:00执行扩容”策略，自动完成CPU、内存、磁盘扩容，并验证服务健康度 避免人工操作失误，提升运维效率
✅ 数据库RDS配置升级 在变更流程中加入“停机窗口验证”、“备份检查”、“性能压测”等步骤，确保升级过程安全 保障数据库高可用，降低宕机风险
✅ Kubernetes集群节点自动伸缩 配置“CPU使用率 > 80% 持续10分钟”自动触发节点扩容，执行后自动验证Pod调度状态 实现弹性伸缩，提升资源利用率
✅ 应用灰度发布与回滚 在变更流程中配置“先发布10%流量 → 验证监控指标 → 全量发布”流程，失败时自动回滚 降低上线风险，保障用户体验
✅ 镜像与配置批量更新 配置“每月1号02:00执行全量镜像更新”策略，自动完成所有ECS实例的镜像升级与系统校验 实现安全、高效、可追溯的批量更新

四、CAC核心操作流程（以华为云CAC为例）

🔧 1. 登录CAC平台

  • 访问地址：https://cac.cloud.huawei.com
  • 使用企业账号登录（需具备 CAC_Super_editor 权限）

📋 2. 创建自动化变更流程（Workflow）

  • 点击「新建变更流程」
  • 选择流程模板（如：ECS扩容、数据库升级、灰度发布）
  • 或从零开始自定义流程，添加以下步骤：
    1. 预检查（如：资源状态是否正常、是否有未完成的变更）
    2. 执行操作（如：调用API扩容ECS、更新RDS配置）
    3. 验证结果（如：检查服务是否恢复、监控指标是否正常）
    4. 通知结果（如：发送邮件/企业微信通知变更完成）
    5. 自动回滚（可选）：当验证失败时，自动执行回滚操作

⚙️ 3. 配置变更策略

  • 在流程中设置以下策略：
    • 执行时间窗口：如“00:00–06:00”或“每周六 02:00–06:00”
    • 最大并发数：如“最多同时执行5个变更”
    • 是否强制执行：如“若审批未通过，是否仍可执行”
    • 影响范围分析：是否在执行前自动分析对上下游系统的影响

📝 4. 设置审批流程与权限

  • 在流程中配置审批节点：
    • 一级审批：开发负责人
    • 二级审批：SRE团队
    • 三级审批：运维主管（可选）
  • 指定审批人或审批组（如：@sre-team）
  • 支持“自动跳过审批”（如：非生产环境变更可自动跳过）

🚀 5. 启动变更流程

  • 点击「立即执行」或「计划执行」
  • 系统将自动进入执行流程：
    • 执行预检查 → 通过则进入执行阶段
    • 执行操作 → 系统自动调用API
    • 验证结果 → 自动检查监控指标或服务状态
    • 若失败 → 自动触发回滚流程
    • 若成功 → 发送通知并生成变更报告

📊 6. 查看变更日志与报告

  • 进入「变更历史」页面
  • 可按以下条件筛选：
    • 变更名称
    • 执行人
    • 时间范围
    • 执行状态（成功 / 失败 / 已回滚）
  • 点击任一条变更记录，可查看：
    • 详细日志（每一步的操作时间、结果、错误信息）
    • 影响范围分析报告
    • 自动回滚记录（如有）
    • 变更前后对比截图（如：资源配置对比）

五、审批与权限管理规范

权限等级 权限说明 适用角色
Super_editor 超级编辑权限 —— 可创建、修改、删除所有变更流程，配置策略与审批流 CAC平台管理员、SRE负责人
Editor 编辑权限 —— 可创建、修改、删除本项目/本部门下的变更流程 项目管理员、开发组长
Viewer 查看权限 —— 可查看所有变更流程与历史记录，但无法修改或执行 所有团队成员
Approver 审批权限 —— 可参与审批流程，对变更申请进行“同意”或“拒绝”操作 SRE、运维主管、开发负责人

✅ 重要提醒：
所有变更流程必须经过 至少一级审批，P1级变更（如：生产环境数据库升级）必须经过三级审批。

六、最佳实践建议

实践 说明
✅ 变更流程“模板化” 将常见变更（如：ECS扩容、数据库升级）封装为可复用的“变更模板”，提升效率
✅ 变更前“影响范围分析”必做 在执行变更前，必须启用“影响范围分析”功能，避免“牵一发而动全身”
✅ 变更失败“自动回滚”必配置 任何涉及生产环境的变更，必须配置“自动回滚”策略，确保系统可快速恢复
✅ 变更报告定期分析 每月组织一次“变更复盘会”，分析变更成功率、失败原因、耗时分布，持续优化流程
✅ 非生产环境可“自动跳过审批” 在测试、预发环境，可配置“自动跳过审批”策略，提升迭代效率

七、常见问题与解决方案（FAQ）

问题 原因 解决方案
❓ 变更流程执行失败，提示“资源忙” 该资源正在被其他变更占用，或存在未完成的变更任务
❓ 审批流程未收到通知 审批人未正确配置通知方式（如：未绑定企业微信）
❓ 自动回滚失败，系统无法恢复 回滚脚本或策略配置错误，或目标资源已不可用
❓ 变更执行时间与策略设定不符 策略配置中时间窗口设置错误，或系统时间不同步
